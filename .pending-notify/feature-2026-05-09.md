*Feature Built — 2026-05-09*

Agent Belief Heatmap
MiroShark simulations now have a per-agent stance grid visualization. Instead of just seeing aggregate belief drift (73% bullish), researchers can see exactly how each individual agent voted every single round — revealing whether consensus emerged through a rapid cascade, slow drift, or a persistent holdout cluster that never joined.

Why this matters:
The aggregate drift chart summarizes; the heatmap is the evidence. Until now, individual agent trajectories were invisible unless you read through raw post logs. For researchers doing social simulation analysis, the difference between "consensus formed gradually" and "half the agents flipped simultaneously in round 5" is the difference between a meaningful finding and an artifact. This was the #1 highest-impact idea from yesterday's repo-actions analysis, directly addressing the individual-level data gap that separates MiroShark as an illustration tool from MiroShark as a primary research instrument.

What was built:
- backend/app/api/simulation.py: New GET /belief-heatmap endpoint. Reads trajectory.json, resolves agent IDs to profile names, computes per-agent per-round stance (YES/NO/NEUTRAL using the same ±0.2 threshold as all other surfaces), detects first flip round and mass flip events.
- frontend/src/components/HeatmapView.vue: CSS grid visualization — each row is an agent, each column is a round, colored by stance (green/red/grey). Sort controls (influence rank, first flip, name). Compact mode auto-activates for >20 agents. Sticky headers for scroll navigation. Hover tooltips with exact values.
- frontend/src/components/Step3Simulation.vue: New Heatmap toggle button in the simulation toolbar alongside Drift, Network, and Demographics.
- backend/tests/test_unit_heatmap.py: 16 offline unit tests covering stance classification, flip detection, sort ordering, name resolution, carry-forward logic, mass flip calculation, and structural assertions.
- backend/openapi.yaml + docs: BeliefHeatmap schema, endpoint path declaration, bilingual API + FEATURES docs.

How it works:
The endpoint reads the existing trajectory.json (same data the drift chart uses) and resolves agent IDs to display names via reddit_profiles.json. For each agent x round, it averages belief positions across topics and classifies stance using the same ±0.2 threshold every other surface uses. Agents are sorted by influence rank (from the leaderboard scorer), first stance-flip round, or name. The frontend renders a pure CSS grid — no canvas needed — with 16x16px cells (12x12px in compact mode). A summary line reports how many agents changed stance and whether a mass flip (>=50% of agents flipping in one round) occurred.
