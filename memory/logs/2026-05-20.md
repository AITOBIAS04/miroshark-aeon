### What was built
Named, ordered lists of published simulations — a curation primitive for citation bundles, research tours, and community arguments.

**Backend (547 LoC):**
- `backend/app/services/collections.py`: Pure-stdlib CRUD service with atomic JSON writes, path-traversal validation, dedup, title/desc caps, sim removal propagation
- `backend/app/api/collections.py`: 5 REST endpoints (POST/GET/GET/:id/PATCH/DELETE) at /api/collections, admin-token gated mutations, simulation summaries with consensus data
- `GET /api/simulation/:id/collections`: Cross-link endpoint showing which collections contain a sim
- OpenAPI spec: Collections tag + 6 schemas (CollectionMeta, Create/Update request, List/Detail/Single envelope)
- `backend/tests/test_unit_collections.py`: 22 unit tests

**Frontend (666 LoC):**
- `CollectionsView.vue`: Gallery of public collections with scenario previews, sim counts, dark-theme design
- `CollectionDetailView.vue`: Ordered sim list with rank badges, consensus bars, scenario excerpts, share-URL copy
- `/collections` and `/collections/:id` routes in Vue Router
- Collections chip on /explore gallery for discovery
- `frontend/src/api/collections.js`: API helper module
