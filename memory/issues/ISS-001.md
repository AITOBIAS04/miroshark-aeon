## Summary

Every scheduled skill has been failing since 2026-04-16 with the same error:

```
{"result":"Not logged in · Please run /login","is_error":true,"total_cost_usd":0,"usage":{"input_tokens":0,...}}
```

The Claude CLI exits immediately without making any API calls — pure auth failure.

## Evidence

- Last successful run for most skills: 2026-04-15
- cron-state.json shows 270–313 consecutive failures per skill, all with 0 tokens used
- No memory logs exist for 2026-04-16 through 2026-04-30 (15-day gap)
- Three full batches of all-skill failures on 2026-04-30 (04:03, 06:37, 09:01 UTC)
- Heartbeat at 10:09 UTC 2026-04-30 succeeded — auth appears restored

## Affected Skills

All enabled skills: token-report, fetch-tweets, repo-pulse, push-recap, project-lens,
repo-actions, repo-article, self-improve, weekly-shiplog, hyperstitions-ideas, feature,
heartbeat, memory-flush, skill-leaderboard

## Required Fix

Renew the `ANTHROPIC_API_KEY` or `CLAUDE_CODE_OAUTH_TOKEN` secret in the GitHub repository
Actions secrets (Settings → Secrets → Actions). The current heartbeat run succeeding suggests
this may have already been done — monitor next scheduled runs to confirm resolution.
